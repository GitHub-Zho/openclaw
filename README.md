# OpenClaw Workspace Backup & Privacy Rules

> 目标：让未来任何 AI/人类在不接触隐私密钥的前提下，快速恢复当前架构与部署流程。

## 1) Push 总原则（隐私优先）

- **禁止上传任何密钥/令牌/口令**：PAT、API Key、JWT、Cookie、Session、SSH 私钥、`.env` 明文。
- **禁止上传可反向识别个人隐私的数据**：手机号、邮箱全值、身份证件、精确住址、私聊原文。
- **允许上传“可执行但脱敏”的知识资产**：
  - 架构说明
  - 部署步骤
  - 运行脚本
  - 回滚流程
  - 故障排查手册

## 2) 脱敏规则（模糊化）

所有会被 AI 读取的文件必须使用以下格式：

- Token/Key 一律写成：`<SECRET_TOKEN>` / `<API_KEY>`
- 邮箱写成：`u***@example.com`
- 手机写成：`+1-***-***-1234`
- Webhook/回调链接保留域名，路径打码：
  - `https://api.example.com/<REDACTED_PATH>`

## 3) 允许进入仓库的“可恢复文件”

必须保留并维护：

1. **架构总览**（当前系统结构、目录、依赖、服务关系）
2. **部署手册**（命令可直接执行，但依赖环境变量注入）
3. **回滚手册**（如何恢复到上一个稳定版本）
4. **故障排查手册**（常见报错 -> 验证命令 -> 修复动作）
5. **Session/Memory 归纳**（仅策略、流程、结论，不含原始敏感对话）

## 4) 禁止进入仓库的内容

- `.env` / `.env.*` 明文
- 私钥、证书、授权码截图
- 含完整个人数据的聊天记录导出
- 第三方平台后台截图（带账号信息）

## 5) 部署可接管规范（给下一位 AI）

所有部署文档必须满足：

- 命令可复制执行（如 `npm run build`、`vercel --prod`）
- 敏感值通过环境变量名表达，不写明文：
  - `VERCEL_TOKEN`
  - `FIGMA_TOKEN`
  - `GITHUB_TOKEN`
- 每个流程附带“成功校验”命令与期望输出
- 每个流程附带“失败回退”步骤

## 6) 备份目标（灾难恢复）

如果当前 AI 会话丢失，新的 AI 应该仅靠仓库即可恢复：

- 当前网站架构与路由规则
- 中英双语策略与语言切换规则
- test/main 分支发布流程
- Vercel 部署与保护策略排查路径
- 常见问题修复顺序

## 7) Push 前检查清单（必须执行）

1. `git diff` 检查是否出现 token / email / phone 明文
2. 搜索敏感词：`token|secret|password|cookie|session|private key`
3. 确认只提交“可恢复知识”和必要代码
4. 提交信息标注范围：`docs|deploy|arch|fix`

## 8) 仓库职责边界

- `ExploreChina2026`：**仅网站代码（site-only）**
- `openclaw`：**AI 运行规则、架构文档、流程资产、脱敏备份知识**
