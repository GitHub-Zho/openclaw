
## Lesson Learned — Website Redesign Requests
- Jo flagged a repeated issue: multiple "redesign" iterations still felt like incremental tweaks.
- Root mistake: preserving too much previous IA/layout while changing visual styling.
- Prevention rule:
  1) If user says "from scratch" / "重新设计", rebuild page IA and structure end-to-end.
  2) Replace major page files (home/trip/handbook) rather than micro-patching sections.
  3) Explicitly report what was rebuilt (IA changes, section order, component structure) to avoid ambiguity.
- Added this rule to `skills/web-brand-builder/SKILL.md` and `references/quality-checklist.md`.
