# 2026-03-09 Visual Coupling Improvement (3 Iterations)

## User complaint
- Text layout acceptable, but text-image coupling was poor.
- Images were semantically weak/misaligned and made site feel AI-generated.

## Root-cause hypothesis
1. Image selection emphasized availability over semantic fit.
2. Hero and city visuals lacked narrative alignment with route content.
3. Section design treated images as decoration instead of evidence/support.

## Iteration 1 — Semantic correction
- Replaced Shanghai image with skyline-specific landmark photo.
- Switched homepage hero to stronger route-relevant visual context.

## Iteration 2 — Text-image coupling structure
- Added "What each city adds" section pairing each image with city-specific value text.
- Converted image usage from decorative strip to explanatory layout.

## Iteration 3 — Attention path polish
- Improved handbook intent block with direct trip-flow CTA.
- Changed handbook hero-side image to Beijing landmark context for narrative closure.

## Learning update
- For travel pages, every image must justify one sentence of decision-relevant meaning.
- Add QA criterion: "Image-semantic alignment" for hero + city sections.
