# 2026-03-09 Web Redesign Iteration Log (Main + Learning)

## Objective
Address user feedback that website looked AI-generated, visually uncoordinated, and insufficiently human-crafted.

## Iteration V1 (baseline assessment)
- Finding: layout was structurally better than initial versions, but visual rhythm and content emphasis felt generic.
- Key issues:
  - Hero lacked strong editorial voice.
  - Program-decision information was fragmented.
  - Handbook page looked like a data dump, not participant-facing narrative.
- Severity: high (user-perceived quality gap).

## Iteration V2 (structure-first redesign)
- Actions:
  - Rewrote homepage structure: hero -> key stats -> rationale -> city structure.
  - Rewrote trip page with clearer package framing and audience-fit blocks.
  - Rewrote handbook page with top-level route/package summary and policy sections.
- Result: stronger IA and decision clarity.
- Remaining gap: still needed stronger visual polish and typographic cadence.

## Iteration V3 (polish pass)
- Actions:
  - Tuned visual system (background/border contrast, card density, button interaction feel).
  - Improved copy tone to sound less templated, more human editorial.
  - Harmonized section pacing and image framing.
- Result: expected improvement in perceived craftsmanship and coherence.

## Learning updates
- Keep redesign loop explicit: V1 diagnose -> V2 structure -> V3 polish.
- Require a short iteration log for any "quality is poor" complaint.
- Treat user-perceived "AI-generated look" as high-priority UX defect.

- Build quality gate passed.
