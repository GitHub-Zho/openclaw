# UX Principles (Practical + Evidence-backed)

## Source anchors
- NN/g usability heuristics (visibility, consistency, error recovery, minimalism).
- WCAG 2.2 quick reference (accessibility baseline).
- Google Core Web Vitals guidance (LCP/INP/CLS targets).
- GOV.UK writing guidance (plain language, meet user need).

## Typography hierarchy
- Keep one explicit scale across pages.
- Recommended baseline:
  - H1: ~2.4-3.0x body
  - H2: ~1.7-2.1x body
  - H3: ~1.3-1.5x body
  - Body: 16px baseline
  - Label/meta: 12-14px
- Avoid random weight changes between sections.

## Spacing rhythm
- Use tokenized spacing (8pt grid): 4/8/12/16/20/24/32/40.
- Section gaps > card internal gaps.
- Avoid arbitrary inline spacing unless exceptional.

## Minimalist language
- One neutral palette + one accent (or two max).
- Prefer 1px border + subtle shadow over heavy decoration.
- Use one radius system for cards/buttons.

## Image system
- Keep one visual family per page.
- Normalize tone: similar saturation, contrast, and brightness.
- Treat images as content support, not decoration.

## Interaction clarity
- One clear primary CTA per section.
- Secondary CTA must not compete with primary.
- Keep nav labels literal and short.
- FAQ should answer real signup objections.

## Accessibility baseline (do not skip)
- Semantic heading order.
- Descriptive alt text.
- Keyboard navigable interactive elements.
- Adequate contrast for text and controls.
