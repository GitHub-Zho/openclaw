# Pre-Launch Quality Checklist

## Visual system
- [ ] H1/H2/H3/body/label scale consistent across all pages
- [ ] Spacing follows the same token system
- [ ] Card radius/border/shadow language consistent
- [ ] Image tone is visually consistent

## UX and conversion
- [ ] Primary CTA visible above the fold
- [ ] Registration CTA repeated near bottom
- [ ] Navigation labels are clear and literal
- [ ] FAQ answers top objections
- [ ] Key actions reachable within 2-3 clicks
- [ ] If "from scratch" was requested, IA structure was actually rebuilt (not only style tweaks)

## Content
- [ ] Value proposition clear in first screen
- [ ] Pricing/inclusions/exclusions are explicit
- [ ] No internal workspace/tool links shown to end users
- [ ] No dynamic/random image endpoints in production pages (e.g., source.unsplash keyword URLs)
- [ ] Copy is concise and plain-language

## Accessibility
- [ ] Semantic heading order (h1 -> h2 -> h3)
- [ ] Alt text on meaningful images
- [ ] Keyboard access for interactive controls
- [ ] Contrast is acceptable for text and controls

## Performance and reliability
- [ ] Images compressed and sized correctly
- [ ] Build passes locally
- [ ] Core Web Vitals targets considered (LCP < 2.5s, INP < 200ms, CLS < 0.1)
- [ ] Deployed routes verified in production
