# Web Design Reference

## Core principles to apply
- Clear hierarchy: users should understand the page in one quick scan.
- Progressive disclosure: summary first, details after intent is shown.
- Predictable navigation: short labels and stable positions.
- Conversion clarity: primary CTA above fold and near decision points.

## Practical structure (landing/program sites)
1. Hero: value proposition + primary CTA
2. Trust/proof: metrics, social proof, partner/credibility cues
3. Core details: what, for whom, how it works
4. Pricing/inclusions near CTA
5. FAQ for objections
6. Final CTA

## Interaction logic
- Keep critical actions within 2-3 clicks.
- Give immediate feedback after interaction.
- Use consistent button treatment for primary vs secondary actions.

## Accessibility and performance baseline
- Semantic heading order.
- Descriptive alt text.
- Keyboard-friendly interactive controls.
- Adequate contrast.
- Optimize image size and loading strategy.

## Evidence anchors
- Nielsen Norman usability heuristics.
- WCAG 2.2 quick reference.
- Core Web Vitals targets (LCP, INP, CLS).
