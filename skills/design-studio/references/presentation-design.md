# Presentation Design Reference (PPT / Keynote / Slides)

## Story architecture
- Open with one-line thesis.
- Sequence: context → problem/opportunity → approach → evidence → ask/next step.
- One key message per slide.

## Slide composition
- Prioritize hierarchy and whitespace.
- Keep text short; convert paragraphs into bullets or visual chunks.
- Use CRAP principles:
  - Contrast (focus)
  - Repetition (consistency)
  - Alignment (order)
  - Proximity (group related info)

## Typography and density
- Limit to 1-2 font families.
- Keep strong title/body contrast.
- Avoid overly light weights for body text.
- Keep a consistent margin and baseline grid.

## Visuals
- Use images/charts that directly support the point.
- Keep visual style consistent across slides (tone, icon style, chart palette).
- Remove decorative visuals that do not add meaning.

## Presenter usability
- Slide should be understandable in 3-5 seconds.
- If a slide requires long explanation, split it.
- End with explicit action: decision, approval, signup, or next milestone.

## Evidence anchors
- CRAP design fundamentals (contrast/repetition/alignment/proximity).
- Presentation Zen style principles (simplicity, restraint, clarity).
