# Design Studio

Use this skill for cross-format design work (web + slides), not only coding.

## 1) Clarify design objective
- Confirm output format: website, single page, app screen, PPT deck, mixed.
- Confirm audience, context, and success metric (conversion, clarity, persuasion, teaching).
- Confirm constraints: brand colors/fonts, deadline, language, device/aspect ratio.

## 2) Build a design system first
- Define typography levels (title/subtitle/body/label).
- Define spacing tokens (8pt grid preferred).
- Define color roles (base, text, accent, semantic states).
- Define component language (buttons, cards, tags, table style, icon style).

## 3) Information architecture and flow
- Organize content by decision order, not by source document order.
- For web: map click flow (entry → trust → detail → action).
- For slides: map story flow (context → tension/problem → proposal → evidence → next step).
- Keep one primary action per section/page/slide when possible.

## 4) Compose layouts
- Use hierarchy + whitespace before decoration.
- Apply CRAP principles (Contrast, Repetition, Alignment, Proximity).
- Keep elements aligned to grid; avoid arbitrary offsets.
- Keep visual rhythm consistent across sections/slides.

## 5) Format-specific rules
- If web-focused, read `references/web-design.md`.
- If presentation-focused, read `references/presentation-design.md`.
- For benchmark examples, read `references/case-library.md`.
- If both, combine shared tokens and adapt per medium.

## 6) Quality review
- Check visual consistency (type, spacing, density).
- Check comprehension speed (can user grasp key point in 3-5 seconds).
- Check accessibility baseline (contrast, readable sizes, clear labels).
- Check redundancy/noise and remove non-essential elements.

## 7) Handoff readiness
- Provide final structure + editable source guidance.
- Provide a quick changelog of what was improved and why.
- If implemented in code, include deployment or preview verification.
